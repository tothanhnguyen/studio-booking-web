Tại sao Modular Monolith?

Thay vì microservices (phức tạp) hay monolith truyền thống (rối), Modular Monolith giữ mọi thứ trong 1 deployable unit nhưng tổ chức code theo module độc lập. Dễ refactor, dễ test, dễ hiểu boundary.

Quy tắc giao tiếp

  • Module chỉ import application layer của module khác
  • Không import trực tiếp infrastructure/ hay domain/
  • Cross-module dependency qua interfaces (Repository pattern)
  • src/lib/ là shared kernel — dùng chung cho mọi module

Shared Kernel (src/lib/)

  • db/prisma — Singleton PrismaClient
  • supabase/ — Auth client (browser, server, middleware)
  • security/ — Guest token generation
  • env/ — Type-safe environment vars (Zod)
  • money/ — VND currency formatting
  • time/ — Timezone-aware date utils

Presentation Layer

  • React Server Components (RSC) cho rendering
  • Server Actions ('use server') cho mutations
  • API Routes cho REST endpoints
  • Không chứa business logic

Application Layer

  • Orchestrate use cases (createBooking, getAvailableSlots)
  • Định nghĩa Repository interfaces
  • Input validation bằng Zod schemas
  • Factory functions cho Dependency Injection

Domain Layer

  • Pure functions — không side effects
  • Business rules: canTransitionBooking()
  • Slot generation: generateAvailableSlots()
  • Domain types & enums
  • Không import bất kỳ infrastructure nào

Infrastructure Layer

  • Implements repository interfaces
  • Prisma queries & mutations
  • Database locking (booking-lock.ts)
  • Chỉ được import bởi application layer

Middleware — Request Pipeline

Next.js middleware chạy ở Edge Runtime trước mọi request. MowStudio dùng middleware để refresh Supabase session. Auth guard nằm ở layout level, không phải middleware.

proxy.ts — Entry Point

File src/proxy.ts là Next.js middleware entry. Nó export function proxy() và config matcher. Mọi request (trừ static files) đều đi qua đây.

  • Matcher loại trừ: _next/static, _next/image, favicon.ico, images
  • Chỉ làm 1 việc: gọi updateSession()

updateSession() — Cookie Refresh

Function trong lib/supabase/middleware.ts. Tạo Supabase server client với cookie handler để:

  • Đọc cookies từ request
  • Gọi supabase.auth.getUser() để refresh token
  • Set lại cookies vào response nếu token được refresh
  • Đảm bảo session không bị expired giữa các requests

Layout Guards — Auth Protection

Thay vì dùng middleware cho auth, MowStudio dùng Layout-level guards:

  • admin/layout.tsx gọi requireRole("ADMIN")
  • UnauthenticatedError → redirect đến login
  • ForbiddenError → trả 404 (ẩn sự tồn tại)
  • Pattern này cho phép RSC rendering với full Node.js runtime

Supabase Auth Flow

  • browser.ts — Client component dùng Supabase browser client
  • server.ts — Server component dùng cookie-based server client
  • middleware.ts — Edge Runtime dùng cho session refresh
  • OAuth callback tại /auth/callback

Booking Flow — Luồng đặt lịch End-to-End

Từ khi khách chọn dịch vụ đến khi booking được xác nhận. Sử dụng Hold pattern với timeout, guest token cho truy cập không cần đăng nhập, và state machine cho booking lifecycle.

Hold Pattern

  • Booking tạo với status PENDING_PAYMENT
  • holdExpiresAt — thời gian giữ chỗ
  • Nếu không thanh toán trước deadline → EXPIRED
  • UI hiển thị countdown (HoldCountdown component)

Guest Access

  • Không bắt buộc đăng nhập để đặt lịch
  • guestAccessTokenHash lưu trong DB
  • Raw token set vào httpOnly cookie
  • Cookie path = /booking/:id (scoped)

Booking Types

  • ROOM_ONLY — Thanh toán → Xác nhận ngay
  • ASSISTED — Thanh toán → Chờ Admin duyệt
  • State machine trong booking-policy.ts kiểm soát mọi transition

Concurrency Control

  • Advisory lock (booking-lock.ts) ngăn double booking
  • Transaction để đảm bảo atomicity
  • Overlap check trước khi insert
  • Idempotency key cho payments
